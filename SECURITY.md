# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for security-sensitive reports. Instead,
contact the maintainer privately through the GitHub security advisory workflow
or the contact method listed on the maintainer profile.

Include:

- Affected version or commit.
- Steps to reproduce.
- Potential impact.
- Any suggested mitigation.

## Scope

Security reports may include dependency risk, unsafe file handling, data leakage,
or behavior that could mislead users into treating research diagnostics as
validated operational decisions.

## Non-Security Issues

Scientific disagreements, documentation corrections, feature requests, and
ordinary bugs should use GitHub issues.

## Supported Versions

The `main` branch is the active research branch. Released tags, if added, should
document their support status.
